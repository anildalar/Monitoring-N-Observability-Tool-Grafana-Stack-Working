apply

The main difference is:

kubectl apply -f → apply raw Kubernetes manifests directly
kubectl apply -k → apply manifests processed by Kustomize


kubectl kustomize overlays/dev
kubectl apply -k overlays/dev


kubectl apply -k overlays/dev
kubectl apply -k base

Kustomize:

reads base
applies patches
generates final YAML
kubectl applies result


kubectl kustomize base
